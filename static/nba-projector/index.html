How the Models Work

Global models
· Naive linear: final = current × 48 / minute. Over-reacts early.
· Linear regression: final ≈ a·minute + b·current + c·team_ppg + d.
· Polynomial deg 2/3/4 + ridge: Adds interaction & curvature terms (e.g., minute²·current, current³). One global polynomial over the full 0–48 minute domain.

Piecewise models — split the time domain into K segments and fit a separate polynomial in each.
· Piecewise LINEAR — 2 / 3 / 4 segments at uniform knots: Halves ([24]), thirds ([16, 32]), and quarters ([12, 24, 36]).
· Piecewise POLY-2 / POLY-3 — 4 segments (quarters): Quadratic and cubic per quarter, capturing within-quarter curvature.
· Piecewise — data-driven knots: Brute-force knot search picking breakpoints that minimize MAE.

Adjustments applied on top of the model output
· Home-court advantage (HCA): +2.5 pts added to the home team's projected final. Long-run NBA HCA estimate.
· History window: the team's team_ppg input is set from the chosen prior — recent-5, weighted, or full 14-season — letting you tune how reactive the projection is to current-roster strength vs long-run norm.

Team PPG priors come from 33,316 real NBA team-game finals (2010–2024) at github.com/NocturneBear/NBA-Data-2010-2024.

Data Refresh Pipeline

Files live under /nba-projector/. The training pipeline is at /nba-projector/pipeline/build_models.py and refreshes via /nba-projector/pipeline/update.sh:
  1. Pulls the latest NBA team-game CSV from the source GitHub repo.
  2. Recomputes per-team priors (recent 5, weighted, overall).
  3. Simulates minute-level trajectories from the new PPG distribution.
  4. Refits all 17 models, writes models.js + data/models.json.
  5. Commits + pushes; Vercel rebuilds the page automatically.
Recommended cadence: weekly during the season, monthly off-season. See /nba-projector/pipeline/README.md for cron / GitHub Actions setup.
